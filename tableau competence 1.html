Autoévaluation de la Compétence 1

Présentation des SAÉ 1.01 et 2.01


Durant cette compétence, j'ai travaillé sur deux projets de développement d'applications Java complémentaires. En S1 (SAÉ 1.01), j'ai conçu avec Guillaume le jeu de la frise chronologique avec une interface web et un mode multijoueur via WebSocket. En S2 (SAÉ 2.01), j'ai participé en équipe de 4 au développement d'un jeu de labyrinthe en 2D avec système de combat, monstres intelligents, inventaire, objets, et progression multi-niveaux à travers 5 itérations de développement agile.


Mes principales réalisations :


  • ✔️ Jeu de frise chronologique avec interface console et web (S1)
  • ✔️ Mode multijoueur temps réel via WebSocket (S1)
  • ✔️ Web scraping automatique d'images historiques (S1)
  • ✔️ Interface HTML/CSS avec Tailwind CSS (S1)
  • ✔️ Jeu de labyrinthe 2D avec système de combat complet (S2)
  • ✔️ Monstres avec IA variable et déplacements automatiques (S2)
  • ✔️ Système d'inventaire et gestion d'objets/armes/soins (S2)
  • ✔️ Labyrinthe multi-étages avec objectif d'amulette (S2)
  • ✔️ Diagrammes UML rigoureux (classes, séquence) pour chaque itération (S2)
  • ✔️ Travail collaboratif Git en équipe de 4 personnes (S2)
  • ✔️ Développement agile avec 5 itérations successives (S2)
  • ✔️ Sprites et interface graphique JavaFX (S2)

Jeu frise chronologique S1 et Jeu labyrinthe S2

Les thèmes abordés incluent le développement Java orienté objet, la conception d'interfaces utilisateur (console, web, JavaFX), la communication réseau, la modélisation UML, le travail collaboratif Git, l'implémentation d'algorithmes dans un jeu, et la gestion de projets avec itérations successives.

AC 1 Implémenter des conceptions simples

100%

 J'ai excellé dans l'implémentation de conceptions simples sur les deux SAÉ. En S1, Guillaume et moi avons réalisé la base du jeu de frise chronologique en moins de 4 heures, puis ajouté des fonctionnalités avancées (WebSocket, interface web, web scraping). En S2, j'ai implémenté efficacement les fonctionnalités assignées à chaque itération : système de vie et cases piégées (itération 1), objets et inventaire (itération 2), monstres intelligents (itération 3), combat et armes (itération 4), puis optimisations finales (itération 5). Mon code était toujours bien structuré, commenté et respectait les spécifications. Cette progression montre ma capacité à transformer rapidement des concepts en code fonctionnel, que ce soit en binôme ou en équipe de 4.
AC 2 Élaborer des conceptions

100%

 Ma capacité d'élaboration de conceptions s'est remarquablement développée entre S1 et S2. En S1, nous avons anticipé l'architecture modulaire pour faciliter l'ajout du multijoueur et de l'interface web. En S2, j'ai participé à la conception rigoureuse avec diagrammes UML complets : diagrammes de classes (Entité, Cases, Items, StrategieDeplacement) et diagrammes de séquence détaillés pour chaque fonctionnalité (combat, inventaire, déplacement monstres, etc.). Notre groupe a produit plus de 30 diagrammes UML sur 5 itérations, planifiant méthodiquement chaque ajout (Factory pattern pour les déplacements, interfaces pour les monstres, composition pour l'inventaire). Cette approche rigoureuse nous a permis de développer un jeu complexe sans retours majeurs.
AC 3 Faire des essais et évaluer les résultats en regard des spécifications

90%

 Mon approche des tests s'est perfectionnée entre S1 et S2. En S1, nous avons testé rigoureusement notre jeu avec une banque de tests unitaires et des retours utilisateurs de camarades, mais les tests du multijoueur restaient limités. En S2, notre équipe a adopté une démarche de tests systématique : tests unitaires à chaque itération, tests d'intégration pour les interactions complexes (combat + inventaire + déplacement), tests utilisateurs pour l'ergonomie JavaFX, et surtout validation Git collaborative avec code review entre chaque membre du groupe. Chaque fonctionnalité était testée avant merge sur la branche principale. Cette approche rigoureuse nous a permis de livrer un jeu stable à chaque itération, bien que certains cas limites du multijoueur de S1 restent non couverts.
AC 4 Développer des interfaces utilisateurs

100%

 J'ai excellé dans le développement d'interfaces utilisateur diversifiées sur les deux SAÉ. En S1, j'ai créé une interface console fonctionnelle, puis développé une interface web moderne avec HTML/Tailwind CSS et WebView Java, incluant du web scraping automatique d'images historiques pour enrichir l'expérience visuelle. En S2, notre équipe a développé une interface JavaFX complète avec sprites visuels, système d'inventaire navigable (touches ZQSD), affichage en temps réel des barres de vie, gestion des écrans de victoire/défaite, et interface intuitive pour toutes les interactions (combat, objets, déplacement). Cette progression démontre ma maîtrise de technologies variées (console, web, JavaFX) et ma capacité à créer des interfaces adaptées aux besoins spécifiques de chaque projet.
