Autoévaluation de la Compétence 3

Présentation des SAÉ 1.03 et 2.03


Durant cette SAÉ (S1), nous avons configurer une machine Linux pour y installer différent logiciel y compris un serveur web. Nous avons de plus eu à coder des scripts bash pour automatiser certaines tâches. L'objectif était de comprendre le fonctionnement d'un système d'exploitation et de mettre en place un environnement de travail. En S2 (SAÉ 2.03), j'ai programmé un serveur web en Java avec fichier de configuration XML, gestion des accès/logs, formulaires HTML, et intégration système Linux via package Debian et service systemD.


Nos principales réalisations :


  • ✔️ Ajouter des utilisateurs et leur attribuer des droits (S1)
  • ✔️ Installation de logiciels sur une machine Linux (S1)
  • ✔️ Configuration d'un serveur web (S1)
  • ✔️ Automatisation de tâches avec des scripts bash (S1)
  • ✔️ Rédaction d'une notice technique pour expliquer les différentes étapes (S1)
  • ✔️ Programmation d'un serveur web Java avec socket TCP (S2)
  • ✔️ Configuration XML avec port, DocumentRoot, sécurité IP (S2)
  • ✔️ Système de logs d'accès et d'erreurs (S2)
  • ✔️ Gestion de formulaires HTML POST/GET (S2)
  • ✔️ Page /status avec état système (mémoire, disque, processus) (S2)
  • ✔️ Package Debian (.deb) et service systemD myweb.service (S2)
  • ✔️ Scripts de démarrage/arrêt et fichier PID (S2)

Compétence 3 : Administrer des systèmes informatiques communicants complexes Niveau de Maîtrise Justification
AC 1 Identifier les différents composants d'un système numérique

95%

 Cette compétence a été démontrée par l'installation et la configuration initiale de l'environnement Linux (S1). En créant des utilisateurs et des groupes, nous avons identifié les rôles spécifiques de chaque composant du système. Par exemple, l'association de l'utilisateur "testuser" au groupe "testgroup" nous a aidé à comprendre comment les permissions sont gérées dans un système Linux. Cette étape nous a permis de comprendre les interactions entre utilisateurs, groupes et le système d'exploitation lui-même. En S2, j'ai approfondi cette compétence en programmant un serveur web qui interagit avec différents composants système : sockets TCP pour la communication réseau, processus Java et gestion des PID, analyse XML avec les APIs javax.xml, accès aux ressources système (mémoire, disque, processus) via les commandes Linux. J'ai compris comment le serveur s'intègre dans l'architecture Linux avec les répertoires standards (/tmp/usr/local, /tmp/etc, /tmp/var/run). Cependant, j'ai parfois eu du mal à bien mettre tous ces éléments ensemble.
AC 2 Utiliser les fonctionnalités de base d'un système multitâches / multiutilisateurs

85%

 L'ajout et la gestion d'utilisateurs sur une machine Linux ont mis en pratique les fonctionnalités fondamentales d'un système multitâches et multiutilisateurs (S1). Par exemple, le script automatisant la création d'utilisateurs est une illustration parfaite de l'utilisation des commandes systèmes comme adduser, la gestion des groupes, et des permissions. Cependant, on aurait dû ajouter des tests plus robustes pour gérer les erreurs ou les entrées incorrectes, pour renforcer la sécurité du système lors de l'execution du script. En S2, j'ai créé un service systemD qui utilise pleinement ces fonctionnalités : le serveur web fonctionne en tant que processus, avec gestion du fichier PID pour le contrôle, logs séparés pour les accès et erreurs, et système de sécurité basé sur les adresses IP. Le service peut être démarré/arrêté par différents utilisateurs selon leurs permissions. J'ai aussi implémenté la gestion des formulaires qui permettent l'interaction multi-utilisateur.
AC 3 Installer et configurer un système d'exploitation et des outils de développement

90%

 L'installation de logiciels comme Apache2, Wireshark et OpenJDK a prouvé la maîtrise de cette compétence (S1). Nous avons rédigé dans la notice technique une documentation détaillée sur les commandes exécutées, démontrant notre compréhension des étapes d'installation et de configuration. Cependant, nous aurions pu ajouter des captures d'écran pour illustrer les résultats de ces commandes, et expliquer plus en détail les options utilisées pour chaque logiciel. En S2, j'ai créé un package Debian (.deb) complet qui automatise l'installation de mon serveur web avec toutes ses dépendances Java, les fichiers de configuration XML, les scripts bash, et l'intégration systemD. Le package installe automatiquement les composants dans les bons répertoires (/tmp/usr/local/sbin, /tmp/etc/myweb, etc.) et configure le service. Néanmoins, je me suis rendu compte que la création du package était plus complexe que prévu, d'autant plus que je travaille sous Windows, et j'ai parfois eu des difficultés avec les dépendances.
AC 4 Configurer un poste de travail dans un réseau d'entreprise

95%

 La configuration du serveur Apache, permettant l'accès local à la documentation technique via l'adresse IP 127.0.0.1, a validé cette compétence (S1). Nous avons également automatisé l'éxecution du serveur au démarrage de la machine, permettant de comprendre la gestion des services via systemctl. En S2, j'ai programmé un serveur web avec configuration réseau avancée : paramétrage du port d'écoute via XML, gestion de la sécurité par plages d'adresses IP (accept/reject), logging des accès pour la surveillance réseau, et gestion des réponses HTTP standard (404, etc.). Le serveur peut servir un site web complet avec gestion des formulaires et page de statut système (/status). Grâce à systemD on a pu avoir un contrôle du service via systemctl start/stop/status myweb.service. La configuration XML était cependant parfois difficile à déboguer.
